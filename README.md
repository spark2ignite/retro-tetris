# Retro Tetris

A classic Tetris game with a retro CRT aesthetic, built entirely in Rust and compiled to WebAssembly. Runs in any modern browser — no dependencies required at runtime.

## Features

- **All 7 tetrominoes** with 7-bag randomisation (fair piece distribution)
- **Ghost piece** showing where the active piece will land
- **Hold piece** — swap the current piece to save for later
- **DAS** (Delayed Auto Shift) for smooth, responsive left/right movement
- **Soft drop**, **hard drop**, and **wall kicks** for rotation
- **Scoring** — 100 / 300 / 500 / 800 × level for 1 / 2 / 3 / 4 lines
- **Level progression** — speeds up every 10 lines
- **Hi-score** persisted in browser `localStorage`
- **Retro CRT aesthetic** — scanlines, phosphor glow, vignette, pixel font
- **Retro sound effects** via Web Audio API

## Running Locally

### Option 1 — Pre-built binary (fastest)

A pre-built release binary is attached to each [GitHub Release](https://github.com/spark2ignite/retro-tetris/releases). Download it, make it executable, and run:

```bash
chmod +x tetris-server
./tetris-server
```

The game opens in your default browser automatically at `http://localhost:8765`.

### Option 2 — Build from source

**Prerequisites:**

- [Rust](https://rustup.rs) (stable)
- [wasm-pack](https://rustwasm.github.io/wasm-pack/installer/)

```bash
# 1. Clone the repo
git clone https://github.com/spark2ignite/retro-tetris.git
cd retro-tetris

# 2. Build the game logic → WebAssembly
wasm-pack build tetris-wasm --target web --out-dir ../web/pkg

# 3. Build the server binary (embeds all web assets)
cargo build --release

# 4. Run
./target/release/tetris-server
```

The browser opens automatically. Press `Ctrl+C` to stop the server.

## How It Works

```
tetris-wasm/        ← Rust library crate
  src/lib.rs        ← all game logic (board, pieces, physics, rendering)
      ↓
  wasm-pack build
      ↓
web/pkg/
  tetris_wasm.js        ← JS glue (wasm-bindgen generated)
  tetris_wasm_bg.wasm   ← compiled game logic (~46 KB)

tetris-server/      ← Rust binary crate
  src/main.rs       ← tiny HTTP server (tiny_http)
      ↓
  cargo build --release
      ↓
target/release/tetris-server   ← single self-contained binary (~716 KB)
```

### Architecture

**Game logic (`tetris-wasm`)** is written in Rust and compiled to WebAssembly using [wasm-bindgen](https://github.com/rustwasm/wasm-bindgen) and [wasm-pack](https://rustwasm.github.io/wasm-pack/). It talks directly to the browser's Canvas 2D API via [web-sys](https://docs.rs/web-sys) bindings — no JavaScript game code at all.

**Server (`tetris-server`)** embeds all three web assets (`index.html`, `tetris_wasm.js`, `tetris_wasm_bg.wasm`) directly into the binary at compile time using Rust's `include_bytes!` macro. At runtime it uses [tiny_http](https://github.com/tiny-http/tiny-http) to serve them and [open](https://docs.rs/open) to launch the browser. No files are read from disk at runtime — the binary is fully self-contained.

## Project Structure

```
retro-tetris/
├── Cargo.toml              # Cargo workspace
├── tetris-wasm/            # Game logic → WASM
│   ├── Cargo.toml
│   └── src/lib.rs
├── tetris-server/          # HTTP server binary
│   ├── Cargo.toml
│   └── src/main.rs
└── web/                    # Web frontend
    ├── index.html          # UI, CSS, WASM loader
    └── pkg/                # Generated by wasm-pack
        ├── tetris_wasm.js
        └── tetris_wasm_bg.wasm
```
